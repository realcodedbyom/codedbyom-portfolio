# Contributing to codedbyom.com Portfolio

Thank you for your interest in contributing to my portfolio website! I appreciate any feedback, suggestions, or improvements you'd like to share.

## How to Contribute

### Reporting Issues
- Use the GitHub Issues tab to report bugs or suggest features
- Provide clear descriptions and steps to reproduce any issues
- Include screenshots if relevant

### Submitting Changes
1. Fork the repository
2. Create a new branch for your feature/fix
3. Make your changes with clear, descriptive commit messages
4. Test your changes thoroughly
5. Submit a pull request with a detailed description

### Code Style
- Follow existing code formatting and conventions
- Use meaningful variable and function names
- Comment complex logic where necessary
- Ensure responsive design principles are maintained

### Areas for Contribution
- Performance optimizations
- Accessibility improvements
- Bug fixes
- Documentation updates
- Design enhancements (with approval)

## Questions?
Feel free to reach out through the contact form on the website or open an issue for discussion.

Thanks for helping make this portfolio better!
